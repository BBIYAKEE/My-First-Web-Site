WEB의 역사

Internet과 Web Internet이 도로라면 Web은 그 위의 자동차 즉, Internet이라는 "전체" 안에 Web, Email, FTP 등 수많은 서비스들이 존재 1960년 인터넷이 등장하고, 30년 후 Web이 등장한다.

Internet

핵 공격에도 버틸 수 있는 강인한 통신 시스템이 필요해서 인터넷이 태어났다. 인터넷은 중심이 없고, 수많은 통신장치들이 분산되어 전화국의 역할을 하므로 하나가 망가져도 사라지지 않음.

Web

Internet은 아주 거대한 기관에서 관리해서 일반 사람들은 잘 몰랐다. Web은 "스위스"에서 태어났다. 스위스 제네바에는 CERN이라는 물리학 연구소가 있는데, LHC(강입자 가속기)가 있는데, 아주 작은 입자를 보는 장치(인류의 현미경). 이 안으로 아주 작은 입자를 고속으로 회전시켜서 입자와 입자를 충돌시키면 어마어마한 충돌이 '힉스 보상'이라는 걸 발생시킴. 12,000명의 어마어마한 지식인들이 모여있는데, 정보의 바다 속에 있는 모습이 오늘날 우리의 모습과 유사 팀 버너스리 라는 사람이 비정규직(프로그래머)으로 취직을 함. "Enquire"이라는 프로그램을 만듦. 이 프로그램이 웹의 전신이 됨. 이런식으로 만들고, 또 만들고 하다가 연구소에 드디어 Internet이 들어왔고, 버너는 인터넷과 본인의 프로그램을 합성하여 웹페이지를 만드는 편집기를 만들고, 세계 최초의 웹브라우저인 WWW을 만듦. 또 웹 서버라는 프로그램을 만들고 info.cern.ch라는 주소를 부여한다. Internet과 Web은 동시에 급속하게 성장하여 오늘날에 이르게 되었다.

이제 전자문서를 전세계의 누구나 이것을 볼 수 있도록 하는 방법을 알아야 한다. 두 대의 컴퓨터가 존재 Web Browser, Web Server. 두 컴퓨터는 인터넷으로 연결되어 있다. Web Server는 http://info.cern.ch라는 주소를 가지고 있다. 웹서버가 설치되어 있는 컴퓨터에는 하드디스크가 있을거고 어느 디렉토리에 index.html이라는 파일이 저장되어 있다. Web Browser에서 http://info.cern.ch/index.html을 쳤을 때 어떤 일이 일어날지 보자. 웹브라우저가 설치되어 있는 컴퓨터는 인터넷을 통해 전기적인 신호를 info.cern.ch에 해당하는 컴퓨터에게 전기적 신호를 보냄. "나는 index.html파일을 원합니다."라는 내용을 포함한 채로. info.cern.ch에 해당되는 컴퓨터에 설치된 Web Server라는 프로그램이 하드디스크에서 index.html을 찾아서 그것을 웹브라우저가 설치되어 있는 컴퓨터에게 전기적신호로 바꿔서 다시 보냄. Web Browser이 설치된 컴퓨터에 index.html의 내용이 보내져서 그것을 읽으면 우리가 그것을 읽을 수 있음. Browser는 정보를 요청(request) -> CLIENT, Server는 응답(response) -> SERVER 역할에 따라서 나누었다.

폰 노이만 "수학은 이해하는 것이 아니라 익숙해지는 것이다." 내 컴퓨터를 전세계와 연결하려면 Web Server를 다루어야겠죠? 2가지 방법이 있는데, 1. 회사에게 맡기는 것 -> Web Hosting *요즘엔 Hosting 대신 Cloud라고도 부름 2. 내 컴퓨터에 까는 것 -> Web Server 컴퓨터가 냉장고처럼 항상 켜져 있어야 하니 관리가 쉽지 않음. Web Server 운영을 위한 소프트웨어까지 빌려주는 업체가 많음. 그 중에 github를 사용해보자. (무료) 원래는 개발자들이 클라우드용보다 자신들의 소스코드를 안전하게 백업하고, 이를 매개로 개발자들끼리 협업하기 위해 고안된 사이트. GitHub에서 repository(저장소) 만들자
